# Change Control

## Zweck
Regelt Aenderungen an Anforderungen, Scope und Governance-Artefakten.

## Regeln
- Aenderungen an FR/NFR muessen in `docs/01_requirements/` dokumentiert werden.
- Traceability ist bei jeder Aenderung zu aktualisieren.
- Aenderungen an Quality/Evaluation muessen in `docs/03_quality/` bzw. `docs/04_evaluation/` gepflegt werden.
- Jede Aenderung aktualisiert `docs/00_overview/PROJECT_STATE.md` und einen Snapshot.

- `docs/01_requirements/NFR.en.md`
