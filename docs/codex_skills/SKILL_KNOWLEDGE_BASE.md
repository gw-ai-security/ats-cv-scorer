# Skill: Knowledge Base Maintenance

## Zweck
Pflege und aktualisiere die Knowledge Base als Single Source of Truth (DE/EN gespiegelt).

## Wann anwenden
- Nach Aenderungen an Requirements, Architektur, Quality, Evaluation, Skills oder Phasenplan.
- Nach jedem neuen Project Snapshot.

## Vorgehen (Schritt-fuer-Schritt)
1) Lies den neuesten Snapshot unter `docs/project_state/`.
2) Aktualisiere die Knowledge Base unter `docs/knowledge_base/de/` und `docs/knowledge_base/en/` gespiegelt.
3) Verlinke vorhandene Artefakte statt Inhalte zu duplizieren.
4) Aktualisiere `Last Updated` in jeder KB-Datei.
5) Pruefe Navigation in `docs/knowledge_base/*/INDEX.md`.

## Qualitaetsregeln
- Keine Annahmen ohne Repo-Beleg.
- DE/EN Inhalte muessen strukturell identisch sein.
- Jede Seite enthaelt Zweck, Kerninhalt, Verweise, Last Updated.

## Verweise
- `docs/knowledge_base/`
- `docs/project_state/`
- `docs/codex_skills/PHASE_PLAN.md`
- `docs/codex_skills/OVERVIEW.md`
