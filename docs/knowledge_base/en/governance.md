# Governance (EN)

## Purpose
Governance rules for requirements, traceability, and change control.

## Core Content
- Definition of Done: criteria for completed requirements.
- Change Control: how requirement and scope changes are documented.
- Review & Acceptance: criteria for formal reviews.
- Requirement Quality: minimum standard for FR/NFR.
- Traceability Rules: mapping requirements to code/tests.
- Scope Guardrails: rules against scope drift.
